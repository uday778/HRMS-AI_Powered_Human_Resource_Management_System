## 🧠 AI Architecture

```
┌─────────────────────────────────────────────────────────────────────┐
│                   GROQ LLM (llama-3.3-70b-versatile)               │
├──────────────────┬──────────────────────────────────────────────────┤
│  Feature         │  Implementation                                   │
├──────────────────┼──────────────────────────────────────────────────┤
│ Resume Scoring   │ PDF → PyMuPDF text → LLM → JSON score + gaps     │
├──────────────────┼──────────────────────────────────────────────────┤
│ HR Chatbot (RAG) │ Docs uploaded → context injected → strict prompt  │
│                  │ "Answer ONLY from context. No hallucination."     │
├──────────────────┼──────────────────────────────────────────────────┤
│ Performance      │ Self + manager ratings → balanced summary +       │
│ Review           │ mismatch detection + 3 development actions        │
├──────────────────┼──────────────────────────────────────────────────┤
│ Leave Analysis   │ Leave history + weekday data → pattern detection  │
├──────────────────┼──────────────────────────────────────────────────┤
│ Offer Letter     │ Candidate details + template → formal letter →    │
│                  │ PDF generated → HTML email sent via SMTP          │
├──────────────────┼──────────────────────────────────────────────────┤
│ HR Insights      │ All KPIs aggregated → executive monthly report    │
└──────────────────┴──────────────────────────────────────────────────┘
```

### 🎯 Key Prompt Design Decision

All AI features return **structured JSON** with explicitly defined schemas — making parsing deterministic and preventing free-text drift in production:

```python
"Return a JSON object with exactly these keys:
{ match_percentage, reasoning, strengths[], gaps[] }
Return ONLY valid JSON, nothing else."
```

## 🚀 How to Run Locally

### Prerequisites
- Python 3.11+
- Node.js 18+
- Free Groq API Key → [console.groq.com](https://console.groq.com)

### 1. Clone the Repository
```bash
git clone https://github.com/uday778/HRMS-AI_Powered_Human_Resource_Management_System.git
cd HRMS-AI_Powered_Human_Resource_Management_System
```

### 2. Backend Setup

```bash
cd backend

# Windows
python -m venv venv
venv\Scripts\activate

# Mac/Linux
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Create upload directories
mkdir -p uploads/employee_docs uploads/resumes uploads/policies uploads/offer_letters
```

Create `backend/.env`:

```env
# Core
GROQ_API_KEY=gsk_your_groq_api_key_here
SECRET_KEY=any_random_32_character_string
HR_EMAIL=hr@arth.com
DATABASE_URL=sqlite:///./hrms.db

# Email (optional — use Mailtrap for testing)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_EMAIL=your_email@gmail.com
SMTP_PASSWORD=your_16char_app_password
```

```bash
# Start backend
uvicorn main:app --reload --port 8000
```

| | URL |
|-|-|
| ✅ Backend | `http://localhost:8000` |
| ✅ API Docs | `http://localhost:8000/docs` |

### 3. Create Admin
```
POST http://localhost:8000/api/auth/seed-admin
```

### 4. Frontend Setup
```bash
cd frontend
npm install
npm run dev
```

✅ Opens at `http://localhost:3000`

### 5. Login
```
Email:    admin@hrms.com
Password: admin123
```

## 🔔 Notification System

Smart notification triggers fire automatically and notify the right people:

| Trigger | Type | Who Gets Notified |
|---------|------|------------------|
| Attrition rate ≥ 15% | 🔴 Critical | Admin |
| 5+ pending leave requests | 🟡 Warning | Manager |
| 30%+ team on leave same day | 🔴 Critical | Manager |
| Open job positions | 🔵 Info | Admin |
| Review cycle with no submissions | 🟡 Warning | Manager |
| New employee joined this week | 🔵 Info | Admin |

Critical notifications also send an email automatically.

- Sent via SMTP with HTML template
