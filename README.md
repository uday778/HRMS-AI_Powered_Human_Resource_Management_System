## 🧠 AI Features Deep Dive

```
┌─────────────────────────────────────────────────────────────────┐
│                    GROQ LLM (llama-3.3-70b)                     │
├─────────────┬──────────────────────────────────────────────────-┤
│  Feature    │  How It Works                                      │
├─────────────┼───────────────────────────────────────────────────┤
│ Resume      │ PDF text extracted → sent to LLM with JD →        │
│ Scoring     │ returns JSON with score, reasoning, gaps           │
├─────────────┼───────────────────────────────────────────────────┤
│ HR Chatbot  │ Policy docs uploaded → context injected into      │
│ (RAG)       │ prompt → "answer ONLY from context" constraint     │
├─────────────┼───────────────────────────────────────────────────┤
│ Performance │ Self + manager data → LLM generates balanced      │
│ Review      │ summary + detects mismatch + suggests actions      │
├─────────────┼───────────────────────────────────────────────────┤
│ Leave       │ Leave history with weekday info → LLM detects     │
│ Analysis    │ patterns like repeated Monday absences             │
├─────────────┼───────────────────────────────────────────────────┤
│ HR Insights │ All KPIs → LLM writes executive monthly report    │
│             │ with highlights, risks, and action items           │
└─────────────┴───────────────────────────────────────────────────┘
```

### Key Prompt Design Decision
All AI features use **structured JSON output prompting** — every prompt explicitly defines the expected JSON schema. This makes parsing deterministic and prevents free-text drift in production.

Example:
```
"Return a JSON object with exactly these keys:
{ match_percentage, reasoning, strengths[], gaps[] }
Return ONLY valid JSON, nothing else."
```

## 🚀 How to Run Locally

### Prerequisites
- Python 3.11+
- Node.js 18+
- Free Groq API Key → [console.groq.com](https://console.groq.com)

### 1. Clone the Repository
```bash
git clone https://github.com/YOUR_USERNAME/hrms.git
cd hrms
```

### 2. Backend Setup
```bash
cd backend

# Create virtual environment
python -m venv venv

# Activate (Windows)
venv\Scripts\activate

# Activate (Mac/Linux)
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Create upload directories
mkdir -p uploads/employee_docs uploads/resumes uploads/policies
```

## Create `backend/.env`

Add the following environment variables:

```env
GROQ_API_KEY=gsk_your_groq_api_key_here
SECRET_KEY=any_random_32_character_string_here
HR_EMAIL=hr@yourcompany.com
```

```bash
# Start backend server
uvicorn main:app --reload --port 8000
```

✅ Backend: `http://localhost:8000`
✅ API Docs: `http://localhost:8000/docs`

### 3. Seed Admin Account
```
POST http://localhost:8000/api/auth/seed-admin
```

### 4. Frontend Setup
```bash
cd frontend
npm install
npm run dev
```

✅ Frontend: `http://localhost:3000`

### 5. Login
```
Email:    admin@hrms.com
Password: admin123
```
